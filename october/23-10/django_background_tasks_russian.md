
# Фоновые задачи и очереди в Django

В этой лекции мы рассмотрим, как использовать фоновые задачи и очереди в Django, включая Celery, Redis и CRON задачи.

## 1. Зачем нужны фоновые задачи?
Фоновые задачи позволяют выполнять длительные операции, такие как отправка электронной почты, обработка файлов или экспорт данных, без замедления основной работы приложения, выполняя эти задачи в других процессах.

## 2. Установка и настройка Celery и Redis
### 2.1 Что такое Celery?
Celery – это библиотека для работы с фоновыми задачами, поддерживающая различные системы очередей. Написана на Python и легко настраивается.

### 2.2 Что такое Redis?
Redis – это база данных с открытым исходным кодом, которая хранит данные в памяти. Celery может использовать Redis для хранения задач.

### Установка:
```bash
pip install celery redis
```

## 3. Создание фоновых задач с Celery
### 3.1 Настройка приложения Celery
Создайте новый файл `celery.py` и добавьте туда следующие настройки:
```python
from celery import Celery

app = Celery('my_project', broker='redis://localhost:6379/0')
app.config_from_object('django.conf:settings', namespace='CELERY')
app.autodiscover_tasks()
```

## 4. Периодические задачи и CRON
### 4.1 Что такое Celery Beat?
Celery Beat – это система планирования, которая позволяет настраивать периодические задачи. Например, можно настроить задачи, которые должны выполняться в определенное время.

### Пример:
```python
from celery import shared_task

@shared_task

    # Код для отправки электронной почты
```
