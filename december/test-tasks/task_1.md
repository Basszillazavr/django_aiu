
# Разработка приложения каталога фильмов

## Описание задания

Ваша задача — разработать веб-приложение с использованием Django, которое отображает каталог фильмов. 

Макет доступен по ссылке: [Макет в Figma](https://www.figma.com/file/AJy3v5JhiyYvyBoShUnUGJ/Frontend---%D0%A2%D0%B5%D1%81%D1%82%D0%BE%D0%B2%D0%BE%D0%B5-%D0%B7%D0%B0%D0%B4%D0%B0%D0%BD%D0%B8%D0%B5?type=design&node-id=0%3A1&mode=design&t=4E349WaaIC2asZVi-1)

---

## Функциональные требования

1. **Главная страница:**
   - Шапка: название сайта, поле поиска, пользователь (фиктивная информация).
   - Поле поиска: ввод поисковой фразы, отображение количества найденных результатов.
   - Контент: карточки фильмов с изображением, именем, годом, ID и типом фильма.
   - Пагинация результатов.

2. **API для получения данных:**
   Используется внешний API [OMDb API](https://www.omdbapi.com/).
   - URL: `https://www.omdbapi.com/?i=tt3896198&apikey=8523cbb8`
   - Метод: `GET`
   - Параметры:
     - `s` (обязательный): строка для поиска.
     - `page` (опциональный): номер страницы (по умолчанию 1).

   **Пример:**
   Поиск фильма с названием "Batman" на 2-й странице:
   `https://www.omdbapi.com/?i=tt3896198&apikey=8523cbb8&s=Batman&page=2`

---

## Требования к реализации

1. **Back-end на Django:**
   - Создайте приложение, которое будет обрабатывать запросы к OMDb API и возвращать результаты через собственный API.
   - Реализуйте кэширование запросов для оптимизации работы (используйте `cache` или сторонние инструменты, такие как Redis).
   - Настройте обработку ошибок, включая случаи, когда результаты не найдены.

2. **Front-end:**
   - Реализуйте клиентскую часть на Django Templates или JavaScript (например, с использованием Vanilla JS или React).
   - Все основные элементы интерфейса должны быть оформлены вручную, без использования UI-фреймворков.

3. **UI и функциональность:**
   - Адаптивность интерфейса для мобильных устройств.
   - Сообщение "Результаты не найдены", если по запросу не найдено фильмов.
   - Индикатор загрузки во время выполнения запросов.
   - Плейсхолдеры для фильмов без изображений.

4. **Карточка фильма:**
   - Изображение.
   - Название.
   - Год выпуска.
   - ID.
   - Тип.

## Дополнительные требования

1. **Время разработки:** 
   - Необходимо зафиксировать время, затраченное на разработку.

2. **Код:**
   - Каждая значимая часть должна быть реализована в виде отдельных Django-приложений или функций.
   - Код выгрузить в репозиторий (GitHub, Bitbucket и т. д.).
